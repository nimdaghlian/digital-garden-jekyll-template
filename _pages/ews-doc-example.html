Tutorial: Create a new risk inquiry

This tutorial describes how you can create a new risk inquiry using the Predict New Account Risk solution as part of the Early Warning Platform.

The Predict New Account Risk solution provides access to 6 unique capabilities that can be used to retrieve a transparent view into a person’s current and prior deposit account history and behavior. Access to these capabilities is based on service bundles, with each bundle containing 2 to 6 Predict New Account Risk capabilities. For more information on the available Predict New Account Risk capabilities and what capabilities are included in each bundle, see Overview

  • The endpoint URL in this tutorial applies to the production environment.
  • If you need generic information about how to code an HTTP request, see Basics of making HTTP requests.
  • The API definition is configured as a downloadable OpenAPI 3.0 file. For more details about the EWS Decision Intelligence Identity Product API, see the API spec.

Use case

This tutorial covers a scenario where, as a staff member of a financial institution, you have a new customer who wants to open an account. You must determine whether you can do business with the customer, and if so, what account services and privileges you can offer them. To do this, you can use the Predict New Account Risk solution with the 49A1012026 bundle to retrieve the First-party Risk Score and Behavioral Attributes for the customer to help inform your decision.

The tutorial describes how to perform this task with the EWS Decision Intelligence Identity Product API.

Prerequisites

Before using the EWS Decision Intelligence Identity Product API, you must:

  1. Be onboarded as an Early Warning client.
    During onboarding, you are assigned a client ID and define how Early Warning can access the public key of the public-private keypair you use to sign your JSON Web Token (JWT) Bearer tokens.

    For information on how to be onboarded as an Early Warning client, see Getting onboarded.

  2. Retrieve an access token.
    The EWS Decision Intelligence Identity Product API endpoints are protected with OAuth 2.0 JWT Bearer grant. To use the API, you must create a JWT Bearer token and use it to get an access token.

    For detailed instructions on how to create a JWT Bearer token and get an access token, see Authentication guide.

Creating a new risk inquiry

This section describes how to use an HTTP POST request to create a new risk inquiry.

Request

Your HTTP request to the EWS Decision Intelligence Identity Product API must contain a path parameter, headers, and a request body.

Request path

Create and send a request using the following path:

POST https://platform.earlywarning.com/predict-new-acct-risk/49A1012026/v1

The path contains the number of your bundle (in this case, 49A1012026). There are different endpoints available for different bundles.

The request must contain the path parameters shown in the following table.

Table: Creating a new risk inquiry – Request path parameters

Authorization Authorization security header Bearer <Access token>

The request also accepts optional headers. For details on all available headers, see the API spec.

Request example

The following example illustrates the request using raw HTTP code:

POST /predict-new-acct-risk/YOUR_BUNDLE/v1 HTTP/1.1 Host: platform.earlywarning.com Content-Type: application/json Accept: application/json Authorization: Bearer YOUR_ACCESS_TOKEN Content-Length: 517 { "clientId": "Authentify_Test", "clientTransactionId": 123456788, "inquiryReason": "bankAccountOpening", "data": { "user": { "name": { "first": "Minnie", "last": "Mouse" }, "dateOfBirth": 19900314, "address": { "streetAddress": "9454 WASHINGTON LOOP", "city": "GOULDSBORO", "region": "PA", "postalCode": 18424 }, "federalId": { "type": "ssn", "value": 933887700 } } } }

Response

If your request succeeds, you receive an HTTP 200 OK response.

If your request fails, you receive an error response. For more information on the possible API errors, see Error handling.
